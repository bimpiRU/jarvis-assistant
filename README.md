# Jarvis Assistant 🤖

Голосовой помощник для Windows на Python с минималистичным Tkinter-интерфейсом. Вдохновлён Джарвисом из фильмов про Железного человека.

## Возможности

- 🎤 **Активация по слову «Джарвис»** — непрерывное прослушивание микрофона
- 🔒 **Офлайн-распознавание речи** через Vosk (не требует интернета)
- 🌐 **Онлайн-fallback** через Google Speech Recognition, если Vosk не уверен
- 🔊 Синтез речи (TTS) через pyttsx3 — работает офлайн
- 🖥️ Графический интерфейс на Tkinter в стиле Jarvis
- 🤖 **Интеграция с Kimi Code CLI** — голосовые запросы к AI-ассистенту
- ⚡ Базовые, системные и сетевые команды
- ✅ **Подтверждение опасных действий** (выключение, перезагрузка, завершение процессов и др.)
- 🔐 **Приватность аудио** — аудио не сохраняется на диск, временные файлы шифруются
- 🚀 Автозагрузка вместе с Windows
- 🔌 Легко расширяется новыми модулями

## Установка

```bash
git clone https://github.com/bimpiRU/jarvis-assistant.git
cd jarvis-assistant
pip install -r requirements.txt
python download_model.py
```

> **Важно:** `download_model.py` скачает ~45 МБ офлайн-модели Vosk для русского языка.

### Разрешение микрофона в Windows

Если Джарвис не реагирует на голос, проверьте разрешение микрофона для Python:

```bash
powershell -ExecutionPolicy Bypass -File fix_mic_permissions.ps1
```

Скрипт откроет доступ микрофона для текущего интерпретатора Python. После запуска перезапустите Джарвиса.

## Запуск

```bash
python main.py
```

### Аргументы командной строки

```bash
python main.py --offline      # Только офлайн-распознавание (Vosk)
python main.py --no-wake      # Отключить активацию по слову
```

Также можно запустить через `start.bat`.

## Использование

1. **Скажите «Джарвис»** — ассистент ответит голосом.
2. Скажите команду.
3. Или зажмите кнопку с микрофоном в окне и говорите.

## Подтверждение опасных действий

Перед потенциально опасными командами (выключение, перезагрузка, сон, очистка корзины, завершение процессов) Джарвис:
1. Голосом просит подтвердить действие.
2. Показывает диалоговое окно с предупреждением.
3. Ждёт голосового ответа: **«подтверждаю» / «да» / «выполняй»** или **«отмена» / «нет» / «не надо»**.

Также есть кнопки **«Подтвердить»** и **«Отменить»** в окне.

## Приватность аудио

- Все аудио-буферы хранятся только в оперативной памяти.
- Временные WAV-файлы шифруются алгоритмом Fernet и удаляются сразу после использования.
- Ключ шифрования генерируется при каждом запуске и не сохраняется.
- История диалога не записывает аудио, только текст.

## Автозагрузка

Чтобы Джарвис запускался вместе с Windows:

```bash
python setup_autostart.py
```

Чтобы отключить:

```bash
python setup_autostart.py off
```

Или голосом: **«Включи автозагрузку»** / **«Выключи автозагрузку»**.

## Голос

По умолчанию Джарвис пытается выбрать **мужской голос**. В Windows обычно доступен мужской голос только для английского языка (Microsoft David). Для качественного мужского русского голоса установите дополнительный языковой пакет Windows:

**Параметры → Время и язык → Язык и регион → Русский → Параметры → Речь → Добавить голос**

После установки Джарвис автоматически выберет русский мужской голос.

## Интеграция с Kimi

Для работы с Kimi должен быть установлен [Kimi Code CLI](https://github.com/moonshotai/kimi-code). Команды вида **«Спроси кими...»** или **«Напиши код...»** откроют новую вкладку Windows Terminal (или PowerShell) с запущенным Kimi и вашим запросом.

## Настройка чувствительности микрофона

В `jarvis/config.py` можно изменить базовые значения:

```python
ENERGY_THRESHOLD = 150   # ниже = чувствительнее
MIC_GAIN_DB = 20         # предусиление в децибелах
NOISE_SUPPRESSION = True # шумоподавление
```

В окне Джарвиса есть **индикатор уровня звука** и **ползунок порога активации**. Если Джарвис не слышит:
1. Проверьте, что индикатор реагирует на голос.
2. Нажмите **«🎙 Проверить микрофон»** — запишет 5 секунд и воспроизведёт. Если вы услышите себя, микрофон работает.
3. Нажмите **«Диагностика»** — покажет RMS, peak и статус микрофона.
4. Понизьте порог активации (ползунок "Порог").
5. Убедитесь, что выбран правильный микрофон.
6. При необходимости увеличьте `MIC_GAIN_DB` в `config.py`.

## Логи

Все события и ошибки пишутся в файл `jarvis.log` в папке проекта. Если что-то не работает — пришлите этот файл.

## Аргументы запуска

```bash
python main.py --offline              # Только офлайн-распознавание
python main.py --no-wake              # Отключить активацию по слову
python main.py --device 1             # Использовать микрофон с индексом 1
python main.py --offline --device 1   # Комбинация
```

Список микрофонов и их индексов можно получить через:

```bash
python -c "from jarvis.mic_diagnostics import print_diagnostics; print_diagnostics()"
```
